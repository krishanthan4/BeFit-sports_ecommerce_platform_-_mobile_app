# Security Policy

## Supported Versions

The following versions of BeFit Sports E-Commerce Platform are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue in the BeFit platform, please follow these steps:

### How to Report

1. **Email**: Send details to the project maintainers (via GitHub issues marked as "Security" or direct contact)
2. **Include**:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact
   - Suggested fix (if available)

### What to Expect

- **Initial Response**: You will receive an acknowledgment within 48 hours
- **Updates**: We will provide updates on the status every 5-7 business days
- **Timeline**: We aim to address critical vulnerabilities within 7-14 days
- **Disclosure**: Once fixed, we will coordinate disclosure timing with you

### Vulnerability Acceptance

**If Accepted:**
- We will work on a fix and include you in the credits (if desired)
- A security patch will be released
- CVE number assigned if applicable

**If Declined:**
- We will provide a detailed explanation
- Alternative mitigations may be suggested if applicable

### Security Best Practices

When deploying BeFit:
- Change default database credentials
- Use HTTPS in production
- Keep dependencies updated
- Enable proper file upload restrictions
- Implement rate limiting on API endpoints
- Use environment variables for sensitive data

Thank you for helping keep BeFit secure!
